# Статус проекта

## Когда использовать

Когда нужно синхронизировать клиента, команду или подрядчиков по текущему состоянию проекта.

## Структура

```markdown
Привет.

Короткий статус по проекту:

1. Что готово
...
2. Что в работе
...
3. Где нужен ответ / решение
...
4. Какой есть риск
...
5. Что предлагаем дальше
...

Следующий контрольный шаг: ...
Срок: ...
Ответственный: ...
```

## Критерий хорошего статуса

После статуса всем понятно:

- что уже сделано;
- что происходит сейчас;
- где нужен ответ;
- какой срок под риском;
- кто делает следующий шаг.
