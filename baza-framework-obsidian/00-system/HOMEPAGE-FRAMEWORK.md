# Фреймворк главной страницы

Главная страница должна помогать человеку быстро войти в базу через свою рабочую ситуацию.

## Задача главной

Человек должен за несколько секунд понять:

1. Что это за база.
2. Для кого она.
3. Какую ситуацию он может здесь решить.
4. Где лежат основные разделы.
5. Какие шаблоны можно забрать сразу.

## Структура главной

### 1. Hero

```markdown
# База знаний продюсера

Для людей, которые ведут проекты, клиентов, команды и учатся отвечать за результат.
```

Дополнительная строка:

```markdown
Здесь собраны рабочие ситуации, принципы, кейсы и шаблоны MASS AGENCY.
```

Кнопки:

- Начать с ситуации
- Открыть разделы
- Забрать шаблоны

### 2. Что случилось?

Главный блок.

```markdown
## Что случилось?

- [[Клиент молчит]]
- [[Срок сдвигается]]
- [[Нужен твёрдый разговор]]
- [[Боюсь написать клиенту]]
- [[Клиент пишет: нужен ролик]]
- [[Правка или новая задача]]
- [[Клиент просит быстро оценить бюджет]]
- [[Нужно предупредить о риске]]
- [[Нужно разобрать ошибку]]
- [[Нужно поставить задачу]]
```

Каждая карточка на сайте должна показывать:

- название;
- одну фразу о ситуации;
- артефакт;
- тег раздела.

### 3. Основные разделы

```markdown
## Основные разделы

### [[Начать с себя]]

Ответственность, страхи, зрелость, личная роль и способность выдерживать сложные решения.

### [[Понять клиента]]

Ожидания, доверие, переговоры, сложные разговоры и право на честный отказ.

### [[Собрать проект]]

Бриф, сроки, бюджет, риски, статусы, роли и контроль результата.

### [[Работать с командой]]

Постановка задач, обратная связь, договорённости, рост и рабочая культура.

### [[Довести до результата]]

Продакшн, постпродакшн, правки, качество, финализация и закрытие.

### [[Управлять агентством]]

Продажи, деньги, процессы, подрядчики, культура и долгосрочные отношения.
```

### 4. Быстрые шаблоны

```markdown
## Шаблоны

- [[Статус проекта]]
- [[Письмо с пониманием задачи]]
- [[Чеклист входа в задачу]]
- [[Сценарий обратной связи]]
- [[Постразбор проекта]]
- [[Письмо о риске]]
```

### 5. Последние обновления

Автоматически подтягивать материалы со свежим `updated`.

Показывать:

- название;
- тип;
- раздел;
- дату обновления.

### 6. Кейсы

```markdown
## Кейсы

Реальные ситуации из агентской работы: ошибки, решения, разговоры, выводы.
```

Показывать только материалы с `visibility: internal` или `public` в зависимости от версии сайта.

### 7. Как пользоваться базой

```markdown
## Как пользоваться

1. Найди ситуацию, похожую на твою.
2. Прочитай короткий разбор.
3. Забери действие или шаблон.
4. Перейди в связанные материалы, если нужно глубже.
```

## Правила для сайта

### Ситуационные карточки

На главной показывать в первую очередь карточки типа:

```yaml
type: situation-card
```

Дополнительный фильтр:

```yaml
status: ready
```

или

```yaml
status: published
```

### Разделы

Разделы собирать по полю:

```yaml
section:
```

### Шаблоны

Блок шаблонов собирать по типу:

```yaml
type: template
```

### Кейсы

Блок кейсов собирать по типу:

```yaml
type: case
```

и учитывать видимость:

```yaml
visibility:
```

## Дизайн главной

Главная должна быть спокойной и быстрой.

### Первый экран

- крупный заголовок;
- короткое объяснение;
- 2–3 действия;
- без длинного вступления.

### Блок «Что случилось?»

Самый заметный блок после hero.

Лучше сетка карточек 2–3 колонки на десктопе и одна колонка на телефоне.

### Блок разделов

Крупные карточки с короткими описаниями.

### Блок шаблонов

Компактный список: человек должен быстро забрать рабочий инструмент.

## Критерий хорошей главной

Главная работает, если человек может за 10 секунд выбрать один из путей:

- открыть похожую ситуацию;
- перейти в нужный раздел;
- забрать шаблон;
- посмотреть последние обновления.
