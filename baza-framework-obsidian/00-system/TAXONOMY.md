# Таксономия базы знаний

Таксономия нужна, чтобы Claude, Obsidian и сайт одинаково понимали материалы: что это за файл, куда он относится, кому помогает и как связан с другими материалами.

## Типы материалов

### situation-card

Карточка рабочей ситуации.

Используется, когда материал начинается с живого напряжения: клиент молчит, команда срывает срок, страшно написать, задача непонятна, проект расползается, нужно отказать или дать обратную связь.

Обязательный результат: действие или артефакт.

### principle

Принцип мышления или управления.

Используется, когда материал объясняет устойчивую идею: ответственность, взрослая позиция, клиент как зеркало, управленческая зрелость, право на твёрдость, решение в рамках ценностей и результата.

Обязательный результат: правило, критерий или новый взгляд.

### process

Рабочий процесс.

Используется для повторяемых действий: вход в задачу, запуск проекта, подготовка статуса, финализация, постразбор, передача в монтаж, работа с правками.

Обязательный результат: последовательность шагов.

### template

Шаблон рабочего артефакта.

Примеры: письмо клиенту, статус проекта, структура брифа, сценарий обратной связи, чеклист входа в задачу, шаблон постразбора, письмо с отказом.

Обязательный результат: готовый текст или структура, которую можно скопировать.

### case

Кейс из практики.

Используется для реальных ситуаций из MASS, клиентских проектов, команды, ошибок и решений.

Обязательный результат: вывод, который можно применить в похожей ситуации.

### checklist

Проверочный список.

Используется, когда человеку нужно пройтись по пунктам перед действием или решением.

Обязательный результат: список проверок.

### guide

Большой разбор темы.

Используется для материалов, где нужна последовательная логика и несколько связанных блоков.

Обязательный результат: маршрут понимания темы.

### glossary

Понятие или термин.

Используется для короткого объяснения слов, которые часто встречаются в базе.

Обязательный результат: ясное определение и пример.

## Разделы

### self

Русское название: «Начать с себя».

Темы: ответственность, зрелость, страх, синдром самозванца, личная роль, самопродюсирование, опора на решение, способность выдерживать сложный разговор.

### client

Русское название: «Понять клиента».

Темы: ожидания, коммуникация, доверие, переговоры, сложные письма, право на отказ, защита результата, понимание пользы клиента.

### project

Русское название: «Собрать проект».

Темы: вход в задачу, бриф, сроки, бюджет, риски, роли, статус, контроль, решения, запуск и закрытие.

### team

Русское название: «Работать с командой».

Темы: постановка задач, обратная связь, договорённости, рабочая зрелость, рост людей, несовпадение по культуре, делегирование, ответственность.

### production

Русское название: «Довести до результата».

Темы: съёмки, монтаж, постпродакшн, финализация, качество, правки, передача материалов, презентация результата.

### money

Русское название: «Деньги».

Темы: стоимость, бюджет, допработы, объяснение цены, маржинальность, финальная ценность, договорённости по оплате.

### agency

Русское название: «Управлять агентством».

Темы: продажи, процессы, культура, найм, подрядчики, база знаний, операционная система, долгосрочные отношения.

## Статусы

- `seed` — есть мысль или короткая заметка.
- `draft` — черновик.
- `review` — материал нужно отредактировать, проверить связи, факты, стиль и пользу.
- `ready` — материал готов к публикации.
- `published` — материал опубликован на сайте.
- `archive` — материал устарел или временно убран из активной базы.

## Уровень доступа

Поле `visibility`.

- `internal` — только для команды MASS.
- `public` — можно публиковать во внешней базе.
- `private` — только для узкого круга. Частные истории, чувствительные кейсы, данные клиентов, деньги, личные ситуации.

## Правила названий

### Для карточек ситуаций

Название должно звучать как рабочая ситуация.

Хорошо:

- Клиент молчит
- Боюсь написать клиенту
- Команда срывает срок
- Нужно дать твёрдую обратную связь
- Проект расползается

### Для процессов

Название должно начинаться с действия или результата.

Хорошо:

- Вход в задачу
- Запуск проекта
- Подготовка статуса
- Финализация ролика
- Постразбор проекта

### Для шаблонов

Название должно объяснять, что человек заберёт.

Хорошо:

- Письмо клиенту о задержке
- Структура статуса проекта
- Чеклист входа в задачу
- Сценарий обратной связи

## Правила связей

У каждой карточки должно быть минимум 3 связи:

1. Похожая ситуация.
2. Процесс или принцип.
3. Шаблон или кейс.

Пример:

```yaml
related:
  - "[[Вход в задачу]]"
  - "[[Статус проекта]]"
  - "[[Боюсь написать клиенту]]"
```

## Правила публикации

На сайт попадают материалы, у которых:

```yaml
status: ready
```

или

```yaml
status: published
```

Материалы с `visibility: private` не публикуются.

Материалы с `visibility: internal` публикуются только во внутренней версии сайта.
