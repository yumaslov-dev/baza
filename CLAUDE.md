# Инструкции для Claude — База знаний команды

Это хранилище — живая командная база знаний. Ты пишешь и поддерживаешь все файлы здесь. Пользователь добавляет сырые знания; ты их структурируешь, связываешь и поддерживаешь актуальность.

## Твоя роль

- Смотри папку `входящие/` — там сырые материалы от команды
- Новые файлы: только в корне `входящие/` (не в `входящие/обработано/`)
- Извлекай ключевую информацию и интегрируй в соответствующие разделы wiki
- Обновляй `_INDEX.md` после каждого изменения
- Добавляй `[[wikilinks]]` между связанными страницами
- Фиксируй противоречия с уже записанным
- После обработки файла — перемести его в `входящие/обработано/`

## Структура разделов

| Папка        | Содержимое                                                                       |
| ------------ | -------------------------------------------------------------------------------- |
| `входящие/`  | Входящие сырые материалы (новые — в корне, обработанные — в `обработано/`)       |
| `команда/`   | История, люди, роли, культура                                                    |
| `процессы/`  | Текущие внутренние и внешние проекты агентства                                   |
| `знания/`    | Продуктовые, отраслевые и общие знания. Рабочие процессы, инструкции, регламенты |
| `docs-src/`  | HTML-документы для сайта (незашифрованные исходники — шифруются автоматически)   |
| `Статьи/`    | Статьи: исходники в `входящие/` (только чтение), результаты работы рядом         |

## ⛔ Правило: Статьи/входящие/

Файлы в папке `Статьи/входящие/` — **неприкосновенны**.

- Никогда не редактировать, не дополнять, не исправлять содержимое этих файлов
- Допустимо только читать их для анализа или создания нового контента на их основе
- Результаты (конспекты, черновики, переработанные тексты) создавать в `Статьи/`, не в `Статьи/входящие/`

## Правила работы

1. **Один файл — одна тема.** Не делай гигантских страниц.
2. **Всегда ставь дату последнего обновления** в начале файла: `_Обновлено: YYYY-MM-DD_`
3. **Используй `[[wikilinks]]`** для ссылок между страницами.
4. **Помечай неточности**: если информация неполная — пиши `> ⚠️ Требует уточнения:`
5. **Формат заголовков**: `# Название`, `## Раздел`, `### Подраздел`

## Как обрабатывать входящие

Когда пользователь говорит "обработай входящие":
1. Найди все файлы в корне `входящие/` (кроме README.md и папки `обработано/`)
2. Для каждого файла: прочитай, определи темы, интегрируй в соответствующие страницы wiki
3. Если подходящей страницы нет — создай новую в нужном разделе
4. Обнови `_INDEX.md`
5. Перемести обработанный файл в `входящие/обработано/`

## Как генерировать документы

Когда просят создать документ (онбординг, инструкцию, презентацию):
1. Собери релевантные страницы wiki
2. Скомпилируй в `документы/название.md`
3. Адаптируй стиль под формат документа (не wiki-стиль)
